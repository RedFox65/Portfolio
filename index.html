Le jeu du Sokoban

L’objectif est de recréer l'environnement du jeu Sokoban, afin qu'il soit jouable par un joueur.

Détails

Le jeu du Sokoban

Projet d'études - BUT 1 (2025-2026)

Description

Dans ce jeu, un personnage doit déplacer des caisses pour les ranger sur des positions cibles. Il peut se déplacer dans les quatre directions et pousser une seule caisse à la fois. L’objectif pour le joueur est de réussir à positionner toutes les caisses sur les cibles avec le moins de déplacements et de poussées possible.

Objectif du projet

Développer le jeu de réflexion Sokoban en langage C. L'objectif étant d'appliquer nos connaissances dans ce langage de programmation, en algorithmique et en gestion de la mémoire.

Technologies utilisées

  • C
  • Git

Défi technique

Problème : La gestion de l'historique des coups pour permettre au joueur d'annuler une action (Ctrl+Z) consommait trop de mémoire.

Solution : J'ai donc implémenté une structure de données en liste chaînée pour stocker uniquement les différences entre chaque déplacement plutôt que de sauvegarder toute la grille.

Compétence liée

Réaliser un développement d'application

Comparaison algorithmique

L’objectif est de rendre autonome et intelligent le Sokoban en lui fournissant un plateau et une suite de déplacements.

Détails

Comparaison algorithmique

Projet d'études - BUT 1 (2025-2026)

Description

Dans sa première version, le programme du jeu Sokoban permettait à un joueur de jouer une partie, c’est-à-dire de se déplacer sur le plateau avec les caisses afin de les mettre au bon endroit.

Objectif du projet

Rendre autonome et intelligent le Sokoban en lui fournissant un plateau et une suite de déplacements à effectuer. Il doit aussi détecter si la suite constitue bien une solution et éliminer de celle-ci toute séquence inutile.

Technologies utilisées

  • C
  • Git

Défi technique

Difficulté : Les séquences inutiles étaient assez difficiles à détecter et à supprimer sans toucher aux déplacements valides.

Solution : Pour cela, il a fallu chercher les combinaisons possibles de déplacements inutiles. Puis ensuite, il a fallu trouver des points identiques dans la façon de les résoudre, avant de rédiger l'algorithme qui permet de supprimer ces séquences inutiles.

Compétence liée

Optimiser des applications informatiques

Création d'une base de Données

Détails

Création d'une base de Données

Projet d'études - BUT 1 (2025-2026)

Description

Nous souhaitons modéliser une partie des données nécessaires à l’étude des candidatures et des admissions dans Parcoursup.
Pour cela, nous disposons d'un diagramme de classes représentant les informations concernant les différentes formations gérées dans Parcoursup.

Objectif du projet

Implanter, peupler puis exploiter une base de données réaliste. Pour cela, nous allons nous appuyer sur les compétences de traduction de modèles de données (UML) et de définition d’un schéma relationnel en SQL.

Technologies utilisées

  • PostgreSQL
  • UML

Défi technique

Difficulté : Bien comprendre le diagramme UML, ses relations et ses multiplicités pour les retranscrire correctement.

Solution : Maîtriser sur le bout des doigts le fonctionnement des diagrammes UML et des schémas relationnels.
